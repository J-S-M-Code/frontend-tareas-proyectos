# SPEC: Agregar Comentario a una Tarea

| Campo                        | Descripción y criterio de calidad                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|:-----------------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Nombre de la feature** | Agregar comentario a una tarea existente. |
| **Descripción general** | Implementar un formulario que permita al usuario añadir un comentario a una tarea específica. El usuario deberá ingresar el texto del comentario y el nombre del autor. |
| **Endpoints involucrados** | - **POST** `/projects/{projectId}/tasks/{taskId}/comments`<br>- **Request Body:** `{ "text": "string", "author": "string" }`<br>- **Response:** `201 Created` con el comentario creado (incluyendo `id` y `createdAt`).<br>- **Errores:** `400 Bad Request` (errores de validación), `404 Not Found` (si la tarea no existe). |
| **Restricciones de negocio** | 1. El campo `text` (contenido del comentario) es estrictamente obligatorio y no puede estar vacío o contener solo espacios en blanco.<br>2. El campo `author` (autor del comentario) es estrictamente obligatorio.<br>3. El comentario debe asociarse a una Tarea que exista en la base de datos. | 
| **Lineamientos técnicos** | - Componentes **Standalone** de Angular 17+.<br>- **Reactive Forms** para manejar las validaciones asíncronas y síncronas en el frontend.<br>- Uso de **Tailwind CSS + PrimeNG** (ej. componentes `InputText` para autor, `Textarea` para el texto, `Button`, `Message`).<br>- **Signals** para manejar el estado local de la vista (ej. `isLoading()`, `successMessage()`, `errorMessage()`).<br>- Actualización del servicio dedicado `TaskService` (o creación de uno si prefieren separarlo) usando `HttpClient`. |
| **Criterios de aceptación** | - **Dado** que el usuario está en el formulario para agregar un comentario, **Cuando** intenta guardar sin completar el texto o el autor, **Entonces** el botón se deshabilita o se muestran mensajes de error de validación bajo los campos.<br>- **Dado** un formulario con datos válidos, **Cuando** el usuario hace clic en guardar, **Entonces** se muestra un indicador de carga, se envía la petición, se muestra un mensaje de éxito indicando que el comentario fue agregado y se resetean los campos del formulario.<br>- **Dado** un `taskId` inválido o inexistente en la URL/contexto, **Cuando** se envía la petición, **Entonces** se intercepta el error `404` del backend y se muestra un mensaje de error indicando que la tarea no fue encontrada. |
