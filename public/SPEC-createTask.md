# SPEC: Crear Tarea

| Campo                        | Descripción y criterio de calidad                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|:-----------------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Nombre de la feature**     | Creacion de trareas para un proyecto|
| **Descripción general**      | Implementar una vista y un formulario que permita al usuario crear una nueva tarea (Task) y asociarla a un proyecto existente. La tarea requerirá un título, una estimación de horas y un responsable.|
| **Endpoints involucrados**   | - **POST**<br>/projects/{projectId}/tasks<br>- **Request Body:**<br> `{ title: string, estimatedHours: number, assignee: string }`<br>-**Response**<br> `201 Created` con el objeto creado<br>-**Errores**<br> `400 Bad Request` (errores de validación)<br>`409 Conflict` (nombre duplicado o regla de negocio).              |
| **Restricciones de negocio** | 1. No se pueden agregar tareas a un proyecto que tenga el estado `CLOSED`<br>2. El campo `estimatedHours` debe ser estrictamente mayor a 0.<br>3. Los campos `title` y `assignee` son de carácter obligatorio.<br>4.El estado inicial de la tarea generada por el sistema será siempre `TODO`          | 
| **Lineamientos técnicos**    | - Componentes **Standalone**.<br>- **Reactive Forms** para manejar las validaciones asíncronas y síncronas en el frontend.<br>- Uso de **Tailwind CSS + PrimeNG** (ej. componentes `InputText`, `Calendar`, `Dropdown`).<br>- **Signals** para manejar el estado de la vista (ej. `isLoading()`, `errorMessage()`).<br>- Servicio dedicado `ProjectService` usando `HttpClient`.|
| **Criterios de aceptación**  | - Dado que el usuario está en el formulario de creación de una tarea para un proyecto activo, cuando completa los campos obligatorios ('title', 'assignee') con datos validos y presiona guardar, entonces el sistema envía la petición al backend, recibe la tarea creada y redirige al listado de tareas del proyecto.<br>- Dado el formulario de creación de tarea, cuando el usuario ingresa el valor 0 en el campo de "Horas estimadas", entonces el botón de guardar se deshabilita y aparece un mensaje indicando que las horas deben ser mayores a 0.<br>- Dado un proyecto con estado CLOSED, cuando el usuario intenta navegar a la pantalla de creación de tareas de ese proyecto, entonces la vista bloquea el formulario y muestra un mensaje informando que "No se pueden agregar tareas a un proyecto cerrado".                |
