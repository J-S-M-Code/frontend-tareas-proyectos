# SPEC: Crear Proyecto

| Campo                        | Descripción y criterio de calidad                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|:-----------------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Nombre de la feature**     | Crear un nuevo Proyecto en el sistema.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Descripción general**      | El usuario podrá registrar un nuevo proyecto ingresando un nombre, la fechas de inicio y la fin, un estado inicial y una descripción.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| **Endpoints involucrados**   | **POST** `/projects`<br>- **Request Body**: <br/>`{ "name": "string", "startDate": "yyyy-MM-dd", "endDate": "yyyy-MM-dd", "status": "PLANNED, ACTIVE, CLOSED", "description": "string" }`<br>- **Response**: <br/>`201 Created` con el objeto creado.<br>- **Errores**: <br/>`400 Bad Request` (errores de validación),<br/> `409 Conflict` (nombre duplicado o regla de negocio).<br>`500` Internal Server Erro (Error no controlado en el código del servidor).                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **Restricciones de negocio** | 1. El nombre del proyecto es obligatorio y debe ser único.<br>2. La fecha de fin (`endDate`) debe ser mayor o igual a la fecha actual (hoy).<br>3. La fecha de fin (`endDate`) debe ser mayor o igual a la fecha de inicio (`startDate`).<br>4. El estado es obligatorio y debe ser uno de los permitidos (`PLANNED`, `ACTIVE`, `CLOSED`).                                                                                                                                                                                                                                                                                                                                                                                     |
| **Lineamientos técnicos**    | - Componentes **Standalone**.<br>- **Reactive Forms** para manejar las validaciones asíncronas y síncronas en el frontend.<br>- Uso de **Tailwind CSS + PrimeNG** (ej. componentes `InputText`, `Calendar`, `Dropdown`), se agregaron las dependencias al `package-lock.json`.<br>- **Signals** para manejar el estado de la vista (ej. `isLoading()`, `errorMessage()`).<br>- Servicio dedicado `ProjectService` usando `HttpClient`.                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **Criterios de aceptación**  | - **Dado** que el usuario ingresa al formulario vacío, **Cuando** intenta enviarlo, **Entonces** se muestran mensajes de error indicando los campos obligatorios.<br>- **Dado** que el usuario ingresa una fecha de fin menor a la de inicio o en el pasado, **Cuando** completa el campo, **Entonces** el formulario marca la fecha como inválida.<br>- **Dado** que el usuario ingresa un nombre de proyecto que ya existe, **Cuando** envía el formulario, **Entonces** la aplicación intercepta el error `409` del backend y muestra un mensaje de "El nombre ya existe".<br>- **Dado** que el usuario ingresa datos válidos, **Cuando** hace clic en guardar, **Entonces** se muestra un indicador de carga, se envía la petición, se muestra el codigo `200`, un mensaje de éxito y se resetea el formulario. |
| **Prompts utilizados**       | Revisar archivo `Promts-createProject.md`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
