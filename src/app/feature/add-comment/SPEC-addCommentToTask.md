# SPEC: Agregar Comentario a Tarea

| Campo                        | Descripción y criterio de calidad                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|:-----------------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Nombre de la feature**     | Agregar comentario a una tarea.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Descripción general**      | Implementar un componente y formulario que permita al usuario registrar comentarios u observaciones de texto en una tarea específica asociada a un proyecto. El comentario requiere el texto y el autor del mismo.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| **Endpoints involucrados**   | - **POST**<br>`/projects/{projectId}/tasks/{taskId}/comments`<br>- **Request Body**: <br>`{ "text": "string", "author": "string" }`<br>- **Response**: <br>`201 Created` con el objeto creado.<br>- **Errores**: <br>`400 Bad Request` (errores de validación).<br>`404 Not Found` (si no existe el proyecto o la tarea).<br>`500 Internal Server Error` (Error no controlado en el código del servidor).                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **Restricciones de negocio** | 1. El comentario debe estar asociado obligatoriamente a una Tarea válida.<br>2. El campo `text` es de carácter obligatorio y no puede estar nulo ni vacío.<br>3. El campo `author` es de carácter obligatorio para identificar quién realiza el aporte y no puede estar vacío.                                                                                                                                                                                                                                                                                                                                                                                     |
| **Lineamientos técnicos**    | - Componentes **Standalone**.<br>- **Reactive Forms** para manejar la entrada de datos y sus validaciones en el frontend.<br>- Uso de **Tailwind CSS + PrimeNG** (ej. componentes `InputTextarea`, `InputText`, `Button`) para mantener la coherencia visual.<br>- **Signals** para manejar el estado de la vista (ej. `isLoading()`, `errorMessage()`, `successMessage()`).<br>- Extensión del servicio `TaskService` usando `HttpClient`.                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **Criterios de aceptación**  | - **Dado** que el usuario ingresó un texto y un autor válidos, **Cuando** hace clic en el botón de comentar, **Entonces** se muestra un indicador de carga, se envía la petición, se limpia el formulario y se muestra un mensaje de éxito.<br>- **Dado** que el formulario tiene el campo de texto o autor vacíos, **Cuando** el usuario intenta enviar el formulario, **Entonces** el botón de envío permanece deshabilitado o se muestran mensajes de error indicando los campos obligatorios.<br>- **Dado** que ocurre un error de conexión o un error en el servidor, **Cuando** se intenta enviar el comentario, **Entonces** el sistema captura el error, oculta el indicador de carga y muestra una alerta visual permitiendo al usuario reintentar. |
| **Prompts utilizados**       | Revisar archivo `Prompts-addCommentToTask.md`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
